macOS-first tmux terminal panel

Cursor 터미널을 tmux 패널로 한 번에 정리하세요.

STP는 Cursor에서 열린 워크트리 터미널을 등록하고, 별도 터미널의 왼쪽 세션 목록과 오른쪽 그리드 패널에서 빠르게 전환할 수 있게 해주는 로컬 도구입니다.

바로 설치하기 명령어 보기
2x2 / 3x3 패널 레이아웃
tmux 세션 보존
Cursor Explorer 연동
stp panel --layout 2x2

>1: api main live

4: <empty> open from Cursor waiting

언제 쓰면 좋은가요?

여러 워크트리와 Cursor 통합 터미널을 동시에 다룰 때 세션 찾기, 전환, 종료를 단순하게 만듭니다.

1

Cursor에서 STP 터미널 열기

확장 프로그램이 STP: tmux 터미널 프로필을 제공하고 macOS 기본 프로필로 설정합니다.

2

tmux 세션 자동 등록

터미널은 stp terminal을 통해 워크스페이스, 윈도우, 터미널 ID를 레지스트리에 저장합니다.

3

패널에서 한 번에 보기

stp panel을 실행하면 왼쪽 세션 목록과 오른쪽 2x2 또는 3x3 그리드가 열립니다.

설치

설치 스크립트는 Rust 바이너리를 빌드하고 Cursor 확장 VSIX를 패키징한 뒤 Cursor에 설치합니다.

1

먼저 dry-run으로 확인

실제 파일을 쓰기 전에 어떤 작업이 실행되는지 확인합니다.

install dry-run 
scripts/install-local.sh --dry-run
2

원하는 prefix로 설치

기본값은 /usr/local입니다. 권한 부담을 줄이려면 홈 디렉터리 아래 prefix를 사용하세요.

custom prefix 
scripts/install-local.sh --prefix "$HOME/.local"
3

Cursor가 stp를 못 찾으면 경로 지정

Cursor를 쉘 밖에서 실행하면 PATH가 다를 수 있습니다. 이때 설정에서 바이너리 절대 경로를 지정합니다.

Cursor settings.json 
"stp.binaryPath": "/absolute/path/to/stp"

Cursor 연동 방식

확장 프로그램은 터미널 프로필, Explorer 뷰, 종료 단축키를 제공합니다.

새 STP 터미널

Cursor에서 STP: tmux 프로필을 선택하면 아래 명령이 실행됩니다.

extension shell args 
stp terminal \
  --workspace <path> \
  --window-id <id> \
  --terminal-id <id> \
  --registry <path> \
  --socket stp-managed

Explorer의 STP Terminals

열려 있는 STP 터미널과 레지스트리의 live 세션을 함께 보여줍니다. 항목을 클릭하면 기존 터미널을 보여주거나 해당 세션에 attach하는 터미널을 새로 엽니다.

STP 터미널을 닫으면 확장은 레지스트리 상태를 detached로 바꾸고, 활성화 및 종료 시 zombie cleanup도 실행합니다.

패널 사용

패널은 tmux 기반입니다. Cursor 터미널에서 만든 live 세션만 오른쪽 그리드에 표시됩니다.

기본 패널 열기

기본 레이아웃은 2x2입니다. stp tui는 같은 패널을 여는 짧은 별칭입니다.

panel 
stp tui
stp panel
stp panel --layout 3x3

왼쪽 세션 목록 클릭

사이드바는 live 세션 수와 주요 키를 먼저 보여줍니다. 이미 오른쪽에 보이는 세션이면 그 pane으로 focus하고, 보이지 않는 세션이면 빈 pane에 엽니다.

패널 종료는 세션 종료가 아닙니다

q는 패널만 닫습니다. 실제 관리 터미널 종료는 prefix K 또는 terminate 명령을 사용합니다.

키맵

패널 조작과 Cursor 터미널 종료 단축키를 한 번에 정리했습니다.

패널만 닫기q
사이드바 세션 focus 또는 열기click
선택된 managed terminal 종료prefix K
Cursor에서 focused STP 터미널 종료cmd+shift+backspace

자주 쓰는 CLI 명령

레지스트리 경로는 기본적으로 ${XDG_STATE_HOME:-~/.local/state}/sexy-terminal-panel/registry.json입니다.

세션에 입력 보내기

터미널 ID 기준으로 입력을 보냅니다. 시각적 슬롯 번호가 아니라 registry의 terminal id를 사용합니다.

send focused 
stp qa-send-focused --terminal-id <id> --text "cargo test"

pane 내용 캡처

tmux pane에서 마지막 줄을 가져옵니다. 기본은 50줄입니다.

capture 
stp qa-capture --terminal-id <id> --lines 80

세션 종료

실수 방지를 위해 --yes가 없으면 종료하지 않습니다.

terminate 
stp terminate --terminal-id <id> --yes

zombie 정리

tmux 세션이 사라졌는데 registry에 남은 live entry를 제거합니다.

cleanup 
stp registry cleanup-zombies --yes

상태 점검

레지스트리 디렉터리 접근성을 확인하고 broker socket을 넘긴 경우 broker 상태도 표시합니다.

doctor 
stp doctor

PTY broker 명령

브로커는 별도 socket을 통해 세션 spawn, input, capture, terminate, list를 처리하는 경로입니다.

broker workflow 
stp broker ensure
stp broker spawn --workspace <path> --window-id <id> --terminal-id <id>
stp broker input --terminal-id <id> --text "pwd"
stp broker capture --terminal-id <id> --lines 50
stp broker list
stp broker terminate --terminal-id <id>
stp broker stop
